5. LinkedList In-place Reversal

Этот паттерн используется для разворота (реверса) части или всего односвязного списка без использования дополнительной памяти.

Когда использовать?

Когда требуется изменить порядок элементов в списке, например, развернуть весь список, развернуть часть между двумя узлами или перевернуть k-группы.

Объяснение

  • Вместо создания нового списка мы меняем ссылки next у существующих узлов.
  • Используются три указателя: prev (предыдущий), curr (текущий), next (следующий).
  • На каждом шаге curr.next перенаправляется на prev, затем все указатели сдвигаются на один узел вперёд.
  • Этот подход позволяет развернуть список за O(n) времени и O(1) памяти.
  • Паттерн часто используется в задачах на разворот части списка, поиска палиндрома, циклических сдвигов и др.
  • Важно не потерять ссылку на следующий узел (next), иначе часть списка станет недоступной.

Пример задачи

Вход: 1 → 2 → 3 → 4 → 5
Выход: 5 → 4 → 3 → 2 → 1

Swift пример

 class ListNode {
    var val: Int
    var next: ListNode?
    init(_ val: Int) { self.val = val; self.next = nil }
}

func reverseList(_ head: ListNode?) -> ListNode? {
    var prev: ListNode? = nil
    var curr = head
    while curr != nil {
        let next = curr?.next
        curr?.next = prev
        prev = curr
        curr = next
    }
    return prev
}

Дополнительный пример

Выход: 1 → 4 → 3 → 2 → 5

Советы и типичные ошибки

  • Не забывайте сохранять ссылку на следующий узел перед изменением curr.next.
  • Проверьте, что head не nil перед началом разворота.
  • В задачах с частичным разворотом аккуратно работайте с границами.
  • После разворота убедитесь, что новый head возвращён правильно.
