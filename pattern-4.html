4. Fast & Slow Pointers

Fast & Slow Pointers (черепаха и заяц) — техника для обнаружения циклов в связанных структурах данных.

Когда использовать?

Когда нужно определить наличие цикла в связном списке или похожей структуре.

Пример задачи

Задача: Определить, есть ли цикл в связном списке.

Объяснение

  • Техника Fast & Slow Pointers (черепаха и заяц) используется для обнаружения циклов в структурах данных, например, в односвязных списках.
  • Один указатель (slow) двигается на один шаг, другой (fast) — на два шага за итерацию.
  • Если в структуре есть цикл, fast рано или поздно догонит slow (они встретятся внутри цикла).
  • Если fast дойдёт до конца (nil), значит цикла нет.
  • Этот метод эффективен по памяти (O(1)), не требует дополнительных структур данных.
  • Применяется не только для поиска циклов, но и для поиска середины списка, определения длины цикла, поиска точки входа в цикл и др.
  • Пример: задача "Happy Number" — fast и slow вычисляют последовательность чисел, пока не встретятся или не достигнут 1.
  • В задачах на массивы fast и slow могут использоваться для поиска дубликатов, палиндромов и других паттернов.

Swift пример

 class ListNode {
    var val: Int
    var next: ListNode?
    init(_ val: Int) { self.val = val; self.next = nil }
}

func hasCycle(_ head: ListNode?) -> Bool {
    var slow = head
    var fast = head
    while fast != nil && fast?.next != nil {
        slow = slow?.next
        fast = fast?.next?.next
        if slow === fast {
            return true
        }
    }
    return false
}

Советы и типичные ошибки

  • Проверьте, что список не пустой перед началом работы.
  • Следите за условиями выхода из цикла (fast?.next != nil).
  • Внимательно относитесь к сравнению ссылок (===) для объектов в Swift.
  • Техника применима не только к спискам, но и к массивам и числовым последовательностям.
