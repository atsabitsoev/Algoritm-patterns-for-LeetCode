15. Greedy

Greedy — жадный паттерн, при котором на каждом шаге выбирается локально оптимальное решение с надеждой получить глобально оптимальный результат. Используется для задач на покрытие, интервалы, размен монет, задачи на оптимизацию.

Когда использовать?

Когда задача допускает локальные решения, которые приводят к глобальному оптимуму, либо когда требуется минимизация/максимизация ресурса, покрытия, количества шагов.

Объяснение

  • Жадные решения часто требуют сортировки входных данных (если это не запрещено).
  • В задачах на интервалы сортируйте по началу или концу интервала для оптимального выбора.
  • В задачах на покрытие (например, размен монет) выбирайте наибольший возможный номинал на каждом шаге.
  • На LeetCode иногда запрещено использовать sort — в этом случае ищите альтернативные подходы (например, подсчёт частот).
  • Проверяйте, что жадный выбор действительно приводит к оптимальному результату (контрпримеры!).

Лучшее решение на LeetCode

На LeetCode жадные решения часто требуют сортировки, но иногда это запрещено условиями задачи — в этом случае ищите альтернативные подходы (например, подсчёт частот). Важно проверять, что жадный выбор действительно приводит к оптимальному результату.

Пример задачи

Вход: coins = [1,2,5], amount = 11
Задача: Найти минимальное количество монет для суммы amount.
Выход: 3 (5+5+1)

Swift пример (жадный подход, если разрешено)

 func coinChange(_ coins: [Int], _ amount: Int) -> Int {
    let coins = coins.sorted(by: >) // Используем sort, если не запрещено
    var amount = amount
    var count = 0
    for coin in coins {
        let use = amount / coin
        count += use
        amount -= use * coin
    }
    return amount == 0 ? count : -1
}

  • В задачах на покрытие выбирайте наибольший возможный элемент.
